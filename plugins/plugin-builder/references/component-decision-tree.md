# 컴포넌트 분류 의사결정 트리

추출된 각 요소에 대해 순서대로 질문하여 컴포넌트 유형을 결정한다.

## 질문 순서

Q1. 이 내용이 모든 상황에 항상 적용되는 불변 규칙인가?
  → YES: **Rule** (alwaysApply: true)
  → NO: Q2로

Q2. 사용자가 명시적으로 "/명령어"로 호출해야 하는 워크플로우인가?
  → YES: **Command**
  → NO: Q3로

Q3. 특정 상황/맥락에서 에이전트가 자동으로 판단하여 로딩하는 지식/절차인가?
  → YES: **Skill**
  → NO: Q4로

Q4. 독립적인 전문가 역할로서 복수 스킬을 조합하여 병렬/순차 실행이 필요한가?
  → YES: **Subagent**
  → NO: Q5로

Q5. 파일 편집, 커밋, 에이전트 종료 등 라이프사이클 이벤트에 자동 반응해야 하는가?
  → YES: **Hook**
  → NO: Q6으로

Q6. 외부 도구/API/데이터소스와 실시간 연결이 필요한가?
  → YES: **MCP Server** 설정
  → NO: 해당 요소는 기존 컴포넌트의 본문에 포함

## 복합 매핑
하나의 요소가 복수 유형에 해당할 수 있다.
예: "캠페인 성과 분석"은 Skill(분석 절차) + MCP(데이터 소스 연동) + Hook(리포트 자동 생성)
