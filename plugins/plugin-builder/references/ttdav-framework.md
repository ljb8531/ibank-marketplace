# T-T-D-A-V Framework

업무 프로세스의 각 단계를 5개 요소로 분해하는 프레임워크.

## 요소 정의

### Trigger (발동 조건)
이 단계를 시작하게 만드는 조건 또는 이벤트.
- 추출 질문: "이 단계는 언제/무엇이 발생하면 시작되나요?"
- 예시: "고객이 캠페인 요청서를 제출하면", "주간 리포트 시간이 되면"

### Think (정보 수집 및 고려)
판단을 위해 수집하고 고려하는 정보.
- 추출 질문: "이 시점에서 어떤 정보를 먼저 확인하나요?"
- 예시: "고객사의 과거 캠페인 성과 데이터를 확인한다", "예산 잔액을 조회한다"

### Decide (의사결정)
의사결정의 기준과 분기 조건.
- 추출 질문: "어떤 기준으로 다음 행동을 결정하나요?"
- 예시: "예산이 1000만원 이상이면 A안, 미만이면 B안", "CTR이 기준치 이하이면 소재 교체"

### Act (실행)
결정 후 수행하는 구체적 행동.
- 추출 질문: "결정 후 실질적으로 무엇을 하나요?"
- 예시: "광고 소재를 생성하여 플랫폼에 등록한다", "분석 리포트를 작성하여 공유한다"

### Verify (검증)
행동 결과의 검증 기준과 방법.
- 추출 질문: "결과를 어떤 기준으로 검수하나요?"
- 예시: "등록된 광고가 정상 노출되는지 확인", "리포트의 수치가 원본 데이터와 일치하는지 교차 검증"

## 사슬 연결 규칙
한 단계의 Verify 출력 → 다음 단계의 Trigger 입력.
이 연결이 전체 프로세스의 흐름도를 형성한다.
연결이 끊기는 지점은 프로세스의 종료점이거나, 누락된 단계가 있는 것이다.
