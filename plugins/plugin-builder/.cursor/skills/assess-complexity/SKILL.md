## 목적
플러그인 규모를 사전에 파악하여 설계 방향을 잡는다.

## 절차
1. `references/complexity-matrix.md`의 평가 매트릭스 참조
2. 프로세스 단계 수, 분기 수, 외부 도구 수, 사용자 수 기준 평가
3. S/M/L 복잡도 판정 및 예상 컴포넌트 수 산출
4. 사용자에게 결과 제시

## 완료 기준
- 복잡도 등급과 예상 컴포넌트 수가 확정됨

## 실패 시
- 경계값일 경우 상위 등급으로 보수적 판정
