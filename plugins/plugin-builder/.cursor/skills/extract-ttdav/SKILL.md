## 목적
업무 프로세스의 각 단계를 구조화된 사고 흐름으로 변환한다.

## 절차
1. Phase 1 수집 데이터에서 업무 단계별로 분리
2. 각 단계에 T-T-D-A-V 5요소 적용 (`references/ttdav-framework.md` 참조)
3. 각 요소를 1~2문장으로 기술
4. 이전 단계의 Verify 출력 → 다음 단계의 Trigger 입력으로 연결

## 완료 기준
- 모든 단계에 5요소가 빈칸 없이 채워짐
- 단계 간 연결이 끊기지 않음

## 실패 시
- 빈 요소가 있으면 사용자에게 해당 부분 재질의
