  각 Act 요소의 실행 방법과 Verify 요소의 검증 기준을
  쌍으로 매핑하고 실패 시 복구 경로를 정의한다.
---

## 목적
행동과 검증의 쌍을 정의하여 플러그인의 에러 핸들링을 설계한다.

## 절차
1. 각 Act 요소에 대해: 실행 도구, 입력, 예상 출력 정의
2. 대응하는 Verify 요소: 검증 기준, 성공 조건 정의
3. 실패 시 복구 경로 매핑: 재시도 / 이전 단계 회귀 / 에스컬레이션
4. 전체를 [Act → 예상산출물 → 검증기준 → 성공경로 → 실패경로] 테이블로 정리

## 완료 기준
- 모든 Act-Verify 쌍이 매핑됨

## 실패 시
- 검증 기준이 불명확하면 사용자에게 "어떻게 확인하시나요?" 질의
