# build-uxui-comparison

UXUI-Analyzer 소속. 서비스별 UXUI 분석 결과를 일관된 기준의 비교표(정량·정성·시각)로 변환하고 패턴을 도출한다.

## 입력

- **services**: 서비스 이름 목록
- **uxui_by_service**: 서비스별 UXUI 분석 결과 (정보 구조, CTA 전략, 시각 디자인 분석 등 1~6단계 분석 결과, 스크린샷 ID 포함)

## 가드레일

1. **비교 항목을 표준화**하여 모든 서비스에 동일하게 적용한다.
2. **정량 / 정성 / 시각** 항목을 분리한 표로 구성한다.
3. 정성 비교는 **미리 정의된 유형 분류에서 선택** + 한 줄 근거를 기록한다.
4. 시각 비교는 스크린샷 기반 관찰에서 **유형 분류 + 근거 스크린샷 ID**를 명시한다.
5. **"확인 안 됨"**은 별도 표기한다.
6. 스크린샷이 없는 서비스의 시각 비교 셀은 **"스크린샷 미확보"**로 표기한다.

## 실행 절차

### Step 1: 정량 비교표 생성

| 서비스 | GNB 항목 수 | 메뉴 깊이 | CTA 유형 | 메인 콘텐츠 섹션 수 |
|--------|------------|----------|---------|---------------------|

- uxui_by_service의 navigation·페이지 분석에서 수치·명칭을 추출해 채운다.
- CTA 유형은 대표 CTA 문구 또는 버튼 라벨(예: "Start Free", "Request Demo")을 한 줄로 기록.

### Step 2: 정성 비교표 생성

**유형 정의(표준 분류에서 선택):**

| 영역 | 유형 옵션 |
|------|----------|
| 정보 구조 | 기능 중심 / 솔루션 중심 / 하이브리드 |
| CTA 전략 | 무료 체험 유도 / 데모 요청 유도 / 가입 유도 / 콘텐츠 유도 |
| 콘텐츠 전략 | 가치 중심 / 기능 중심 / 사례 중심 |
| 톤앤매너 | 전문적·엔터프라이즈 / 친근한·스타트업 / 미니멀·심플 / 정보 밀집 |
| 복잡도 처리 | 단계별 가이드 / 탭·아코디언 / 시각화 / 시나리오 기반 |

| 서비스 | 정보 구조 | CTA 전략 | 콘텐츠 전략 | 톤앤매너 | 복잡도 처리 |
|--------|----------|---------|------------|---------|------------|

- 각 셀에 **유형 + 한 줄 근거**를 기록한다. 별도 근거 테이블을 만들어 셀별 근거를 매핑해도 된다.

### Step 3: 시각 비교표 생성

**유형 정의(표준 분류에서 선택):**

| 영역 | 유형 옵션 |
|------|----------|
| 레이아웃 | 좌우 분할 / 중앙 집중 / 풀와이드 / 카드 그리드 |
| Hero 구성 | 이미지 중심 / 텍스트 중심 / 제품 스크린샷 중심 / 영상 중심 |
| 색상 전략 | 모노톤 / 브랜드 컬러 강조 / 그라데이션 / 멀티컬러 |
| 컴포넌트 스타일 | 라운드 / 샤프 / 미니멀 / 리치 |

| 서비스 | 레이아웃 | Hero 구성 | 색상 전략 | 컴포넌트 스타일 | 참조 스크린샷 |
|--------|---------|----------|----------|----------------|---------------|

- 각 셀에 **유형 + 근거 + 참조 스크린샷 ID**(예: `main_hero`, `main_features_summary`)를 기록한다.
- 스크린샷이 없는 서비스·항목은 **"스크린샷 미확보"**로 표기한다.

### Step 4: 패턴 도출

- **가장 많은 서비스가 선택한 유형** → 업계 공통 패턴 목록
- **소수만 선택한 유형** → 차별화 패턴 목록
- **반복되는 유형 조합** → 트렌드 목록
- 정성 비교와 시각 비교의 **교차 패턴** 도출  
  (예: "솔루션 중심 정보 구조 + 제품 스크린샷 Hero + 전문적 톤" 조합이 엔터프라이즈 타겟에서 반복)

## 출력

1. **정량 비교표** — Step 1 결과
2. **정성 비교표** — 유형 + 근거(테이블 또는 인라인)
3. **시각 비교표** — 유형 + 근거 + 참조 스크린샷 ID (또는 "스크린샷 미확보")
4. **업계 공통 패턴** 목록
5. **차별화 패턴** 목록
6. **유형 조합 트렌드** 목록

### 구조화 출력 (선택)

다운스트림 자동 활용이 필요하면 다음 형태로 요약한다.

```json
{
  "quantitative": { "서비스별 정량 항목" },
  "qualitative": { "서비스별 정성 유형 + evidence" },
  "visual": { "서비스별 시각 유형 + evidence + screenshot_ids" },
  "patterns": {
    "common": ["패턴 설명 목록"],
    "differentiators": ["패턴 설명 목록"],
    "trends": ["유형 조합·트렌드 목록"]
  }
}
```

## 사용 시점

- UXUI Analyzer **분석 7단계**에서 1~6단계 결과를 받은 뒤 1회 호출.
- 서비스 추가 수집·스크린샷 보강 후 **비교표 재생성**이 필요할 때 재호출.
- 벤치마크 리포트의 UXUI 비교 섹션에는 이 스킬이 생성한 정량·정성·시각 비교표와 패턴을 그대로 포함하고, 시각 비교표의 참조 스크린샷 ID로 마크다운 이미지 링크를 매핑한다(예: `![서비스 Hero](../../research/screen/{slug}/main_hero.png)`).
